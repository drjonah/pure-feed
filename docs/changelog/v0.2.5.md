# v0.2.5

## Text-Based NSFW Filtering

- Added keyword/regex text scanning that detects NSFW content in post text
- Posts with explicit text are deleted from the feed (no blur option for text)
- Scans across all supported platforms: X, Reddit, Instagram
- Uses categorized patterns matching the same labels as image filtering (Sexy, Porn, Hentai)
- Toggle in Settings > Text Filtering (off by default)

## Custom Word List

- Users can add their own custom words and phrases to filter
- Custom words are matched case-insensitively alongside the built-in patterns
- Words appear as removable chips in the settings UI
- Custom words section only visible when text filtering is enabled

## Filter Strength UI

- Renamed "Advanced Thresholds" to "Filter Strength"
- Inverted the slider scale so higher values = stronger filtering
- Added hint text explaining the scale
- No change to internal logic or stored settings (existing user settings are preserved)

## CSP Fix

- Moved the dev-only Chrome API mock from an inline script to an external module
- Fixes Content Security Policy violation (`script-src 'self' 'wasm-unsafe-eval'`) that blocked the options page from loading in the extension context

## Stats

- Added "Text filtered" summary card to the Stats tab
- Text-filtered posts tracked separately from image-filtered posts

## Other

- Bumped version to 0.2.5 (manifest.json, package.json)
- Added Incognito setup instructions to README
- Updated README with text filtering documentation
- Updated About page default settings to include text filtering fields
